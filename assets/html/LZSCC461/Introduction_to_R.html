R Markdown & Reproducibility

Reproducible research

  • For every analysis your aim should be to have a script which reads in the raw data, prepares the variables, performs the analysis, produces the plots
  • You should be able to rerun this from beginning to end without bugs
  • This is so you can prove to others, and more importantly yourself, what you have done
  • Ideally you should also have saved the output file (.Rout or knitr report) generated from your script file
  • Do not just type commands in at the command prompt and copy the output

The traditional workflow

  1. Write code in an R script
  2. Run code to get results i.e.Â numbers and plots
  3. Copy the results to somewhere else e.g.Â MS Word or LaTeX
  4. Insert the plots manually into the report
  5. Figure something wrong, back to R script
  6. Forget to save (some of) the updated results
  7. The report contains some old and some new results
  8. Others cannot reproduce your analysis

The reproducible way

  • Ideally, we have a “notebook” file that contains:

    • chunks of code
    • any accompanying text

  • The document reporting the analysis is then generated dynamically, meaning that when the code/data changes, the results change accordingly

  • This generated document contains the following interweaved:

    • the results i.e.Â numbers and plots
    • the accompanying text explaining these results
    • any code required

R Markdown notebooks

  • We can add a new chunk by clicking the *Insert Chunk* button on the toolbar or by pressing *Ctrl+Alt+I*.

  • Type your R code for any analysis required within the chunk

R Markdown notebooks

  • When you save the notebook, an HTML file containing the code and output will be saved alongside it (click the *Preview* button or press *Ctrl+Shift+K* to preview the HTML file).

  • This HTML file is for you to preview the results, and is not the generated document

R Markdown notebooks

  • To generate the actual document, click the black arrow next to *Preview* (or *Knit* when you reopen the file)

  • Choose *Knit to HTML* or *Knit to PDF*. The generated document will pop up and automatically saved in your working directory

  • If you knit to PDF, you also need LaTeX installed on your system
